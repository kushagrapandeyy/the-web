## Overview

Clothing store checkout redesigning was for a class during my time at ASU, where I was responsible for building just a checkout page for a clothing brand as part of a HCI course allowing me to understand the value of creating seamless designs in the interfaces that I code.

## Role

**UX Designer - class assignment.** Focused exclusively on the checkout and cart experience.

## Tools

- Figma - interaction design, high-fidelity mockups

## Problem

The original checkout experience had three core problems:

1. **Accidental item removal** - the cart delete action was prominently placed and offered no undo, leading to user frustration
2. **Limited payment options** - only card payment was supported, with a poorly formatted input layout
3. **Unnecessary steps** - the multi-page checkout flow added cognitive load and increased drop-off risk between steps

## Key Design Changes

**Cart Item Management** - Moved the delete action behind a swipe gesture with a timed undo option. This prevents accidental removals while keeping the interaction discoverable.

**Payment Methods** - Expanded the payment section with clear visual hierarchy: card, digital wallet (Apple Pay / Google Pay), and buy-now-pay-later options presented as distinct, equally weighted options - not buried in a dropdown.

**Single-Page Checkout** - Condensed the flow from multiple steps into one scrollable page. Each section (items, shipping, payment, confirmation) is visible at a glance, reducing cognitive load and time-to-completion.

**Persistent Order Summary** - An order summary sidebar follows the user throughout checkout so they always have visibility into what they're purchasing, eliminating the need to navigate back.

## Outcome

A redesigned checkout flow in Figma demonstrating reduced friction from cart to confirmation - covering cart management, single-page checkout layout, expanded payment methods, and the order confirmation state.
