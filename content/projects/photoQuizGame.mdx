## Overview

WordFinder is a game I just made as a personal project, which is currently under development to make it functional, monetize it and then add it on the app store. It was kind of a pet project I did to boost my app interface designing skills as well as android and apple application development skills using Swift and Kotlin.

## Role

**Designer - personal project.** Responsible for concept, UI layout, and wireframes.

## Tools

- Figma - low-fidelity wireframes and interface states

## Design Focus

### Game Board Layout

The core design challenge was making a letter grid visually clean and easy to scan. Decisions made:

- Monospaced, evenly spaced grid cells for predictable eye movement across rows
- High contrast between unselected letters and highlighted found-word sequences
- Letter size calibrated so the full grid fits in a single viewport without scrolling

### Interface States

Four key UI states were designed end-to-end:

1. **Start Screen** - topic selection and difficulty, sets expectations before committing
2. **Active Game** - live grid with word list sidebar, score, and timer visible simultaneously
3. **Word Found** - momentary highlight animation on the found word, then list item checked off
4. **Game Complete** - summary of score, time taken, and option to replay or share

### Visual Restraint

The interface is deliberately minimal - a neutral palette with a single accent colour for selection highlights. This keeps the letter grid the clear focal point and prevents UI chrome from competing with gameplay.

## Outcome

A set of low-fidelity wireframes covering all four core interface states - demonstrating a complete, coherent game experience from start screen through completion.
