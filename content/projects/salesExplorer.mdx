Raw data is boring. Numbers don't tell a story until you actually explore them. This project is about turning sales data into something you can interact with and learn from.

**The Web Dashboard**

Built in React with D3.js and Vega-Lite for visualization. Filterable views across product categories, regions, and time windows. Multiple chart types: line charts, bar comparisons, heat maps. Responsive tooltips showing detailed breakdowns on hover. Real-time filtering without page reloads because nobody wants to wait for the page to reload.

**The Analysis Pipeline**

Python notebooks for exploratory analysis. Used Pandas for data transformation, Matplotlib and Seaborn for publication-quality charts. Did statistical testing to separate signal from noise.

**What the Data Revealed**

Seasonal patterns in chocolate sales—spike before holidays, dip in summer. Regional differences in product preferences. Premium products focused in wealthy regions. Customer cohort behavior: repeat buyers have 3x higher lifetime value than one-time purchasers.

The last insight is the important one. Once you see it, it changes how you think about the business. Not "how do we sell more chocolate" but "how do we turn one-time buyers into repeat customers."

**Why This Matters**

The technical skills here bridge software and business. As an engineer, I can take raw data and make it understandable. I can identify patterns. I can present findings clearly. That's valuable in any company—you're not just building features, you're helping people understand their own business.
