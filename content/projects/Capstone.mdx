Dating apps are increasingly commoditized—everyone's building the same swipe interface. This capstone explores what happens when you combine thoughtful UX with a scalable, intelligent backend. The goal: a system where good matches actually feel intentional, not algorithmic.

## **The Backend Architecture**

**FastAPI + Microservice Design**
Core service running on async ASGI, structured around four domain models: authentication, profiles, messaging, and recommendations. Each service exposes clean REST endpoints with schema validation via Pydantic. Request/response serialization handles profile images through Cloudinary URLs to avoid serving large blobs.

**Database: PostgreSQL with Strategic Design**
Schema optimized for the actual query patterns:
- `profiles` table indexed on (user_id, active) for feed queries
- `preferences` normalized to separate table for efficient filtering
- `matches` with composite index on (user_id, created_at) for pagination
- `messages` partitioned by user_id for scalability

Result: 200ms feed queries instead of 5+ seconds from a naive schema.

**Matching Algorithm**
Two-stage filtering:
1. Explicit constraints: age ± range, distance ≤ radius, interest intersection ≥ 3
2. Compatibility scoring: weighted overlap on preferences + interaction history signal

Weights calibrated from user feedback—highest engagement comes from users with 60-80% preference alignment, not 100%.

**Real-time Messaging**
WebSocket connection pooling per user. Messages persisted to PostgreSQL with cursor-based pagination. Delivery status tracked (sent/read) to enable read receipts without polling.

## **The Frontend**

**React + TypeScript**
State management through context for global auth, user profile, and recommendations. Feed rendering uses virtualization for infinite scroll without DOM explosion. Profile cards animate in with staggered entrance—keeps UX lively without motion sickness.

**Expo Mobile**
Native experience on iOS/Android. Share auth tokens between web/mobile. Image picker integrates with Cloudinary upload widget for instant preview.

**Optimistic Updates**
Swipe actions update UI instantly before server confirmation. Failures gracefully revert with user notification. Messaging shows "sending..." state with automatic retry on connection restore.

## **What Actually Mattered**

Database normalization and query optimization compounds. Bad schema = every feature slow. Good schema = features that feel instant. Strategic indexing cut response times by 25x.

API design shapes frontend velocity. Clean endpoints grouped by domain made frontend development 3x faster than if the API exposed raw query results.

Async processing everywhere—uploads, image resizing, recommendation computation. Connection pooling prevents cascading failures. These aren't flashy but they're what separates "works" from "works at scale."
